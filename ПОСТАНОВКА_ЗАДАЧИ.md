# ПОСТАНОВКА ЗАДАЧИ

Информационная система управления данными высшего учебного заведения представляет собой программу для автоматизации процессов учета и управления данными образовательного процесса в университете: ведение учета факультетов и кафедр, управление учебными группами и студентами, учет преподавательского состава, управление учебными дисциплинами, распределение учебной нагрузки преподавателей, учет дипломных работ и ведение успеваемости студентов с автоматическим расчетом оценок [1].

Необходимо реализовать: возможность добавления, редактирования и удаления всех основных сущностей системы (факультеты, кафедры, группы, студенты, преподаватели, дисциплины, учебная нагрузка, дипломные работы), возможность группировки данных для удобного отображения иерархических структур, возможность управления профилями студентов и преподавателей с отображением связанных данных, возможность автоматического вычисления курса дисциплины на основе семестра, возможность автоматического определения текущего семестра на основе времени года и курса группы, возможность автоматического расчета итоговых оценок на основе баллов за семестр и экзамен/зачет с учетом формы контроля, возможность валидации всех вводимых данных с проверкой корректности и соблюдения бизнес-правил, возможность сохранения и загрузки всех данных в текстовые файлы для обеспечения персистентности между сеансами работы [2].

Взаимодействие с данными осуществляется через графический интерфейс, построенный на основе паттерна MVVM (Model-View-ViewModel). Все сущности представлены классами моделей данных, которые инкапсулируют информацию и предоставляют методы для сериализации и десериализации. Каждая сущность имеет уникальный идентификатор для установления связей между объектами. Сохранение и загрузка состояния системы происходит с помощью методов сериализации и десериализации соответственно, которые преобразуют объекты в строки с разделителями и обратно. При работе с дисциплинами автоматически вычисляется курс на основе семестра, что гарантирует соответствие курса и семестра. При открытии профиля студента автоматически определяется текущий семестр на основе текущей даты и курса группы с учетом времени учебного года. При вводе оценок автоматически рассчитывается итоговая оценка на основе суммы баллов за семестр и экзамен/зачет с использованием различных шкал оценивания для экзаменов и зачетов. При добавлении учебной нагрузки проверяется бизнес-правило о количестве преподавателей на определенных типах занятий для одной группы по одной дисциплине. При назначении научного руководителя для дипломной работы проверяется наличие у преподавателя права руководить работами. Все вводимые данные проходят валидацию на соответствие допустимым форматам и диапазонам значений, обязательные поля проверяются на заполненность [3].

При разработке программы необходимо использовать объектно-ориентированное программирование, инкапсуляцию данных через приватные поля и публичные свойства, наследование для создания иерархии классов, полиморфизм через переопределение методов, применение паттерна MVVM для разделения логики и представления, использование реактивного программирования для автоматического обновления интерфейса при изменении данных, реализацию группировки данных для удобного отображения иерархических структур, обеспечение валидации всех вводимых данных с отображением сообщений об ошибках, реализацию алгоритмов автоматического расчета курсов, семестров и оценок, обеспечение персистентности данных через сохранение в текстовые файлы [4].
